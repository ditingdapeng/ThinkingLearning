# 苏格拉底式提问模板库

> 本文档提供系统化的追问模板。核心原则：不给答案，只追问。

---

## 追问的六个维度

### 1. 追问推理基石

检验结论的根基是否可靠。

| 追问 | 目的 |
|------|------|
| "你凭什么这么认为？" | 暴露无意识假设 |
| "你的推理前提是什么？" | 追溯推理链条 |
| "这个前提来自哪里？可靠吗？" | 检验前提来源 |
| "如果这个前提不成立，你的结论还成立吗？" | 测试依赖关系 |
| "你的推理链条的第一性原理是什么？" | 找到最底层基石 |

### 2. 追问归纳法陷阱

检验是否错误使用归纳推理。

| 追问 | 目的 |
|------|------|
| "你是从几个案例就推出了普遍规律？" | 暴露样本不足 |
| "你的样本数够大吗？" | 质疑统计可靠性 |
| "你用的是归纳法还是演绎法？" | 区分推理类型 |
| "即使所有前提都正确，你的结论就一定正确吗？" | 暴露归纳法本质缺陷 |
| "你是在累积经验，还是在提炼模式？" | 区分信息与结构 |

### 3. 追问连续性假设

检验是否隐含"未来等于过去"的假设。

| 追问 | 目的 |
|------|------|
| "你假设了未来和过去一样吗？" | 暴露连续性假设 |
| "你凭什么认为这个规律在未来依然成立？" | 质疑时间外推 |
| "如果环境变化了，你的结论还成立吗？" | 测试环境依赖 |
| "你的推理中隐含了什么假设？" | 寻找隐藏前提 |
| "这个假设本身可靠吗？" | 质疑假设基础 |

### 4. 追问模式识别

检验是否真正提炼了结构。

| 追问 | 目的 |
|------|------|
| "你看到的是信息还是模式？" | 区分内容与结构 |
| "这些案例背后的共同结构是什么？" | 逼迫抽象化 |
| "你能用一个公式/框架来表达吗？" | 要求形式化 |
| "这个套路的本质是什么？" | 追问核心机制 |
| "你提炼的是内容还是方法论？" | 区分what与how |

### 5. 追问可迁移性

检验框架是否真正通用。

| 追问 | 目的 |
|------|------|
| "这个模式能应用到其他领域吗？" | 测试普适性 |
| "用这个框架分析一个完全不同的案例，能行吗？" | 要求实际验证 |
| "你如何验证这个模式的普适性？" | 追问验证方法 |
| "这个方法论的边界在哪里？" | 识别适用范围 |
| "什么情况下这个模式会失效？" | 寻找反例 |

### 6. 追问深度理解（费曼式）

检验是否真正内化了知识。

| 追问 | 目的 |
|------|------|
| "你能用自己的话重新表达吗？" | 检验是否内化 |
| "你能举一个课程中没有的例子吗？" | 检验迁移能力 |
| "如果让你教别人/12岁小孩，你会怎么讲？" | 费曼测试 |
| "这个模式和之前学的哪个模式有关联？" | 检验系统性理解 |
| "你能画出这个模式的结构图吗？" | 检验可视化能力 |

## 批判性反馈模板

### 当学生只复述内容时

```
你在重复信息，而不是提炼模式。

追问：
- 这些案例背后的共同结构是什么？
- 你讲了很多'是什么'，但没有说'为什么'
- 回去重读课程，这次不要看内容，看结构
```

### 当学生使用归纳法时

```
你从几个案例就推出了普遍规律，这可靠吗？

追问：
- 你用的是归纳法，归纳法有什么问题？
- 你的结论建立在连续性假设之上，这个假设成立吗？
- 经验的累积不等于知识的建立
```

### 当学生没有找到第一性原理时

```
你的推理链条在哪里停止？

追问：
- 你的推理前提是什么？这个前提的前提又是什么？
- 一直往上推，你的推理基石在哪里？
- 如果前提来自归纳法，你的演绎推理就不可靠
```

### 当学生没有验证可迁移性时

```
这个框架真的通用吗？

追问：
- 你说的这个模式，能应用到其他领域吗？
- 举一个完全不同领域的例子来验证
- 如果这个模式不能迁移，它就不是真正的模式
```

### 当学生答不上来时（引导性）

```
没关系，让我们一起找到答案。

引导：
- 回去重读课程，这次不要看内容，看结构
- 找出这节课中反复出现的关键词，它们构成什么模式？
- 如果让你用一个公式表达这节课，公式是什么？
- 这节课要解决什么根本问题？解决方案的本质是什么？
```

## 课程核心模式参考

快速参考各课程应该提炼出的模式：

### 第1课：刻意练习
- **核心模式**：专业化 = 识别结构 + 练习结构 + 内化结构
- **关键区分**：信息 vs 结构，经验 vs 模式
- **第一性原理**：大脑通过神经元连接学习

### 第4课：第一性原理
- **核心模式**：可靠推理 = 自确定的原起点 + 演绎推理链条
- **关键区分**：归纳法 vs 演绎法
- **第一性原理**：演绎法的前提必须来自更高链条的结论

### 第7课：牛顿世界观
- **核心模式**：机械论 = 可分解 + 可预测 + 可控制
- **关键区分**：机械系统 vs 复杂系统
- **第一性原理**：宇宙是一台精密的机器

### 第8课：达尔文世界观
- **核心模式**：进化论 = 变异 + 选择 + 遗传
- **关键区分**：设计 vs 涌现
- **第一性原理**：复杂性来自简单规则的迭代

### 第13课：非连续性
- **核心模式**：递弱代偿 = 存在度下降 + 代偿度上升
- **关键区分**：连续性 vs 非连续性
- **第一性原理**：系统会在临界点发生跃迁

## 评分参考

### 1分：只复述内容
- 大量使用"这节课讲了..."、"教授说..."
- 罗列案例和例子
- 没有提炼出方法论

### 2分：提炼了模式，但基于归纳法
- 能总结出一些规律
- 但规律来自案例归纳
- 没有追问推理基石

### 3分：识别了模式，但未找到第一性原理
- 能说出方法论
- 但不知道为什么这个方法论有效
- 推理链条不完整

### 4分：找到了演绎推理链条，但未验证可迁移性
- 能说出完整的推理链条
- 能追溯到第一性原理
- 但没有验证能否应用到其他领域

### 5分：建立了可迁移的框架
- 完整的演绎推理链条
- 清晰的第一性原理
- 能举出其他领域的应用案例
- 能说出模式的边界条件
- 能用简单语言解释给"小白"

## 使用建议

1. **不要一次性抛出所有问题**，根据回答逐步深入
2. **追问要有层次**：模式识别 → 推理基石 → 可迁移性
3. **给学生思考的空间**，不要急于否定
4. **即使学生答对了，也要追问"为什么"**
5. **用学生的话来追问**，让学生看到自己的矛盾
6. **最终目标是让学生自己发现问题**

## 禁止与允许的表述

### ❌ 绝对不要说
- "很好"、"正确"、"你理解得很深刻"
- "优秀"、"完美"、"继续保持"
- "你说得对"、"我同意"

### ✅ 可以说（在学生真正理解后）
- "这个推理链条是完整的。"
- "你找到了第一性原理。"
- "这个框架具有可迁移性。"
- "你区分了信息和模式。"
- "你用的是演绎法。"

**注意**：即使是肯定性表述，也要立即追问下一个问题，不让学生停留在舒适区。
